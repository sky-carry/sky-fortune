# Sky Fortune 总体方案与技术选型

> 决策记录：2026-07-02 确定 **A+B 双线路线**——共享一套命理引擎和内容库；A 出海站做付费测算+订阅（主营收），B 境内站做传统文化工具（黄历+起名，不碰付费测算）。分开域名、主体、支付。
> 依据见 [调研总结](../research/00-调研总结与功能规划.md)。

---

## 一、双站定位

| | A 出海站 | B 境内站 |
|---|---|---|
| 定位 | AI 中式命理（八字/紫微/合婚），面向海外华人 + 欧美东南亚玄学用户 | 传统文化工具站：老黄历/择日/节气 + 起名测名 |
| 语言 | 英文为主，繁中/简中次之 | 简体中文 |
| 变现 | 订阅 $9.99/月（每日运势+AI 问答）+ 单次深度报告 $19-49（PDF） | 广告 + 起名服务（268 元档起）|
| 支付 | Stripe | 微信/支付宝商户号，**类目只报"文化服务/起名"**，站内绝不出现付费测算 |
| 可以做 | 八字精批、合婚、紫微、塔罗、AI 问答、"改运"话术也合法 | 黄历、择日、节气、生肖文化、起名、免费娱乐向测名（去承诺化表述）|
| 不可以做 | — | 付费测算/占卜/运势解锁、消灾改运话术、抽签收费 |
| 部署 | Vercel / 海外云 | 境内云 + ICP 备案 |

两站共享：排盘引擎、规则引擎、解读语料库、UI 组件。**代码共享，数据和用户体系不共享**（合规隔离 + PIPL/GDPR 各自处理）。

## 二、技术选型

**全栈 TypeScript**。决定性理由：两大核心开源库 lunar-javascript/lunar-typescript（八字+黄历全套）和 iztro（紫微斗数）都是 JS/TS 原生，引擎层零翻译成本；且两站都重度依赖 SEO，Next.js SSR/SSG 是现成答案。

| 层 | 选型 | 说明 |
|---|---|---|
| Monorepo | pnpm workspaces + Turborepo | 两 app 多 package |
| 前端框架 | Next.js（App Router） | SSR/SSG 做 SEO；词条页静态生成 |
| 多语言 | next-intl（A 站） | en / zh-TW / zh-CN |
| UI | Tailwind + shadcn/ui；react-iztro（紫微星盘图） | 视觉基调：去神秘化"新中式"极简 |
| 历法/排盘 | **lunar-typescript**（八字、黄历、称骨、节气）+ **iztro**（紫微） | 均 MIT；西占后期再引 Swiss Ephemeris（注意 AGPL/商业双许可） |
| 数据库 | Postgres + Drizzle ORM | A 站 Supabase/Neon；B 站境内 RDS |
| 认证 | Auth.js（A 站：email + Google）；B 站先无账号体系 | B 站工具属性，降低个信收集 |
| 支付 | Stripe（A）；微信支付服务商模式（B，仅起名服务） | |
| AI | Claude API（claude-sonnet-5 生成报告，claude-haiku-4-5 做每日运势批量） | 只在 A 站上线；架构上"命盘 JSON + 语料检索 → prompt → 报告"，不裸聊 |
| 内容管理 | 语料以结构化 JSON/MDX 存 repo，后期迁 CMS | 词条（十神/星曜/数理）即 SEO 页面 |

## 三、仓库结构

```
sky-fortune/
├─ apps/
│  ├─ global/        # A 出海站 (Next.js, next-intl, Stripe)
│  └─ cn/            # B 境内站 (Next.js, 黄历+起名)
├─ packages/
│  ├─ engine/        # 核心引擎（纯函数、无 IO、100% 单测）
│  │  ├─ bazi/       #   排盘、十神、五行旺衰、喜用神、大运流年
│  │  ├─ naming/     #   五格三才、81数理、八字补缺、生肖宜忌、起名推荐
│  │  ├─ hehun/      #   合婚打分（生肖/纳音/日柱/用神互补）
│  │  ├─ almanac/    #   黄历宜忌、择日反查、冲煞
│  │  ├─ ziwei/      #   iztro 封装
│  │  └─ misc/       #   称骨、诸葛测字、抽签、梅花易数
│  ├─ content/       # 解读语料库 + 数据资产（康熙笔画、汉字五行、81数理批语、签文、太岁表…）
│  ├─ ai/            # 报告生成 pipeline（prompt 模板 + 语料检索 + LLM 调用）
│  └─ ui/            # 共享组件（命盘卡片、五行图、星盘、结果分享卡）
└─ docs/             # research/ + plan/
```

## 四、引擎层关键技术决策（流派选定，一次定死写进测试）

| 分歧点 | 决定 | 理由 |
|---|---|---|
| 年/月柱换界 | 立春/节气交节时刻（精确到分） | 主流标准，lunar 默认 |
| 晚子时（23:00-24:00） | 日柱用当日、时柱起子时（"晚子时"派） | 与问真八字等主流排盘一致，便于比对 |
| 真太阳时 | 默认关闭，高级选项可开（输入出生城市） | 降低输入门槛；专业用户可开 |
| 五行旺衰 | 加权打分法（天干/地支/藏干/月令系数），阈值判旺弱 | 可程序化、可回归测试 |
| 起名笔画 | 康熙字典繁体笔画 + 部首特殊计数规则 | 行业惯例 |
| 回归基准 | 与元亨利贞、问真八字逐例比对，≥100 个典型生日测试集 | 覆盖立春边界、晚子时、闰月、跨时区 |

## 五、里程碑

**M1 引擎 + 数据资产（先行，双站共用）**
八字排盘/十神/五行/喜用神/大运 → 五格测名+起名推荐 → 黄历宜忌 → 合婚打分；同步采集数据资产（康熙笔画表、汉字五行表、81 数理、太岁表、称骨表）；回归测试集跑通。

**M2 B 境内站上线（先上，简单且验证引擎）**
老黄历（今日宜忌/择日反查）+ 节气生肖内容页 + 测名/起名工具（免费打分 + 付费人工起名服务表单）+ 广告位。全站去承诺化文案审查。ICP 备案（备案周期长，**第一周就启动**）。

**M3 A 出海站 MVP**
英文站：八字排盘（Bazi chart）+ 免费简批 + 付费精批 PDF（LLM 生成，$19-29）+ Stripe + 邮件注册。落地页做 Nebula 式渐进问答（克制版），SEO 词条页（十神/五行/生肖 in English）。

**M4 A 站留存与订阅**
每日运势推送（邮件/PWA push）+ 订阅制 + 八字合婚报告 + AI 问答（限定语料、按次提问包）+ 分享命格卡片。

**M5 扩展**
紫微斗数（iztro）→ 塔罗 → 西方占星（评估 Swiss Ephemeris 商业许可）→ 好友合盘社交。

## 六、风险与对策

1. **B 站起名收款类目**：以"文化创意服务"申请商户号，页面表述为"姓名文化定制服务"，避免"改运""补五行转运"话术 → 上线前过一遍红线清单（research/04 §5.2）。
2. **A 站内容质量**：LLM 报告必须挂在确定性命盘数据上，每份报告附排盘结果，防"什么都能说"的可信度疲劳。
3. **Swiss Ephemeris AGPL**：西占功能上线前决定买商业许可或该模块单独开源；M5 之前不引入。
4. **语料版权**：解读文案自建（可 LLM 辅助初稿 + 人工审校），不抓取现代解签/命书文本。
5. **域名/品牌**：两站不同品牌名，互不外链（合规隔离）；A 站名建议英文原生（待定）。
